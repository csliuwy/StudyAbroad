## 1. 背景与目标

- 希望通过 **`http://studyabroad.martxdata.com/`**（不显式端口，即 **TCP 80**）访问与本地 **`http://localhost:5180`** 同套 StudyTour 前端，并由 **Nginx** 将 **`/api`** 反代到后端 **8080**。
- 过程中涉及：本机安装 Node/npm、后端启动方式、根路径 `Cannot GET /`、域名与 `resolveApiBase`、云上访问失败排查、安全组与 RDP 来源 IP 等。

## 2. 主要问题与结论

### 2.1 浏览器 / `curl` 访问域名 HTTP 超时，但 `ping` 通

- **现象**：`ping studyabroad.martxdata.com`（ICMP）正常；`curl http://121.43.75.225/` 与 `curl http://studyabroad.martxdata.com/` 对 **80** 长时间 **Connection timed out**。
- **结论**：**ICMP 放行 ≠ TCP 80 放行**。超时表示 **SYN 在到达 ECS 上 Nginx 之前被丢弃**（云上策略），不是「Nginx 未监听」——本机已确认 **`0.0.0.0:80` 监听**、**iptables/nft 默认 ACCEPT**。
- **根因（控制台核对后）**：实例实际绑定的安全组（示例 ID **`sg-bp1gtyx1ouyp2uuxs2rw`**）入方向 **未配置 TCP 80（及未配 443）**，仅有 **8080、5180、22、3389、ICMP** 等；故 **HTTP 必失败**，**ping 仍可成功**。
- **处理**：在该安全组 **入方向** 增加 **自定义 TCP 80/80**（若需 HTTPS 再加 **443/443**），来源按需 **`0.0.0.0/0`** 或收紧为固定公网 IP；并确认 **改动的安全组与 ECS 主网卡绑定的一致**。

### 2.2 前端 API 基址与域名

- 前端曾写死 **`http://localhost:8080/api`**，通过域名访问时浏览器会请求用户本机，错误。
- **结论**：使用 **`frontend/src/apiBase.ts`** 的 **`resolveApiBase()`**（`VITE_API_BASE_URL` 优先；开发直连 Vite 非 80 端口时用 **同 hostname:8080**；经 Nginx 的 **80/443** 用同源 **`/api`**；生产构建默认 **`/api`**）。

### 2.3 Nginx 与静态资源路径

- **`root` 指向 `/root/.../frontend/dist`** 时，**`nginx` 进程用户无法读 `/root`**，出现 **500 / Permission denied**。
- **结论**：静态文件部署到 **`/var/www/studyabroad`**（由 `dist` 拷贝），**`chown -R nginx:nginx`**；站点配置见仓库 **`StudyAbroad/deploy/studyabroad.martxdata.com.conf.example`**（含 **`/health`** 反代、**`default_server`** 说明）。
- 更新前端后需同步：`sudo cp -a …/frontend/dist/. /var/www/studyabroad/` 再 **`reload` nginx**。

### 2.4 后端根路径 `Cannot GET /`

- **结论**：Express 未注册 **`GET /`** 时默认返回 **Cannot GET /**；已在后端增加 **`/`** 说明 JSON（以仓库当前代码为准）。

### 2.5 安全组中 **3389** 与 **`114.248.14.110`**

- **`ipconfig` 常见 192.168.x.x**：为**内网地址**，不会出现在「来源 IP」白名单语义里。
- **`114.248.14.110`**：为**公网出口 IP**；规则表示 **仅该公网 IP 可访问 RDP(3389)**，换网络出口后需改规则。
- **22、8080、5180** 等对 **`0.0.0.0/0`** 开放：任意来源可尝试连接；若需加固，可像 3389 一样收窄来源。

### 2.6 本机环境

- 若 **`npm: command not found`**：在 Alibaba Cloud Linux 上已通过 **`dnf install -y nodejs npm`** 解决，并在 **`frontend` / `backend` / `scripts`** 执行 **`npm install`**。

## 3. 待办 / 风险（自检）

- [ ] 安全组已加 **80**（及计划中的 **443**）后，在 PC 上 **`curl -v http://studyabroad.martxdata.com/`** 复测。
